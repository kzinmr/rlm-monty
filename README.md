# RLM-Monty: Recursive Language Models with Monty Sandbox

[日本語](#日本語) | [English](#english)

---

## English

A minimal implementation of Recursive Language Models (RLM) running on the [Monty](https://github.com/pydantic/monty) sandbox for secure code execution.

### What is RLM?

RLM (Recursive Language Model) is an approach to handle long contexts by allowing an LLM to recursively query sub-LLMs. Instead of processing an entire context at once, the model can interactively explore and analyze the context using a REPL environment.

[Link to the paper](https://arxiv.org/abs/2512.24601v1)

[Link to the original blogpost](https://alexzhang13.github.io/blog/2025/rlm/)

### Why Monty?

This implementation uses [Monty](https://github.com/pydantic/monty), a sandboxed Python interpreter written in Rust, for executing code generated by the LLM. Monty provides:

- **Security**: Execute untrusted Python code safely without filesystem, network, or environment access
- **Performance**: Fast startup time (<1μs) and runtime performance similar to CPython
- **Simplicity**: Clean implementation focused on AI-generated code execution

### Installation

```bash
# Using uv (recommended)
uv add rlm-monty

# Or using pip
pip install rlm-monty
```

### Quick Start

1. Copy the example environment file and add your OpenAI API key:

```bash
cp .env-example .env
# Edit .env and add your OPENAI_API_KEY
```

2. Run the example:

```bash
python main.py
```

### Usage

```python
from rlm import RLM_REPL

# Initialize RLM with Monty sandbox
rlm = RLM_REPL(
    model="gpt-4o",              # Main model
    recursive_model="gpt-4o-mini",  # Model for recursive calls
    enable_logging=True,
    max_iterations=10
)

# Use RLM to process a long context
context = "Your long context here..."
query = "What is the answer?"
result = rlm.completion(context=context, query=query)
print(result)
```

### Example: Needle-in-a-Haystack

The `main.py` file demonstrates a classic needle-in-a-haystack problem where a magic number is hidden within 100,000 lines of random text. RLM uses the REPL environment to search through the context and find the answer.

### Code Structure

- `rlm/rlm_repl.py` - Main RLM implementation with REPL environment
- `rlm/repl.py` - Monty-based sandboxed REPL environment
- `rlm/utils/` - Utility functions for LLM calls, prompts, and parsing
- `rlm/logger/` - Colorful logging utilities for debugging

### Differences from rlm-minimal

This implementation differs from the original `rlm-minimal` in the following ways:

1. **Monty Sandbox**: Uses Monty for secure code execution instead of Python's `exec()`
2. **Security**: Code execution is sandboxed with no filesystem, network, or environment access
3. **Performance**: Faster startup time for code execution

## 日本語

[Monty](https://github.com/pydantic/monty)サンドボックス上で動作するRecursive Language Models (RLM)の最小実装です。安全なコード実行環境を提供します。

### RLMとは？

RLM（Recursive Language Model）は、LLMが再帰的にサブLLMをクエリすることで長いコンテキストを処理するアプローチです。コンテキスト全体を一度に処理するのではなく、モデルはREPL環境を使用してコンテキストを対話的に探索・分析できます。

[論文へのリンク](https://arxiv.org/abs/2512.24601v1)

[オリジナルのブログ記事へのリンク](https://alexzhang13.github.io/blog/2025/rlm/)

### なぜMonty？

この実装では、LLMが生成したコードを実行するためにRustで書かれたサンドボックスPythonインタプリタ[Monty](https://github.com/pydantic/monty)を使用しています。Montyは以下を提供します：

- **セキュリティ**: ファイルシステム、ネットワーク、環境変数へのアクセスなしで、信頼できないPythonコードを安全に実行
- **パフォーマンス**: 高速な起動時間（<1μs）とCPythonに類似した実行性能
- **シンプルさ**: AIが生成したコードの実行に焦点を当てたクリーンな実装

### 使い方

```python
from rlm import RLM_REPL

# MontyサンドボックスでRLMを初期化
rlm = RLM_REPL(
    model="gpt-4o",              # メインモデル
    recursive_model="gpt-4o-mini",  # 再帰呼び出し用モデル
    enable_logging=True,
    max_iterations=10
)

# RLMを使用して長いコンテキストを処理
context = "ここに長いコンテキスト..."
query = "答えは何ですか？"
result = rlm.completion(context=context, query=query)
print(result)
```

### 例：干し草の中の針

`main.py`ファイルは、魔法の数字が100,000行のランダムテキストの中に隠されている古典的な「干し草の中の針」問題を示しています。RLMはREPL環境を使用してコンテキストを検索し、答えを見つけます。

### コード構成

- `rlm/rlm_repl.py` - REPL環境を使用したメインのRLM実装
- `rlm/repl.py` - MontyベースのサンドボックスREPL環境
- `rlm/utils/` - LLM呼び出し、プロンプト、解析用のユーティリティ関数
- `rlm/logger/` - デバッグ用のカラフルなロギングユーティリティ

### rlm-minimalとの違い

この実装はオリジナルの`rlm-minimal`と以下の点が異なります：

1. **Montyサンドボックス**: Pythonの`exec()`の代わりにMontyを使用した安全なコード実行
2. **セキュリティ**: ファイルシステム、ネットワーク、環境変数へのアクセスなしでサンドボックス化されたコード実行
3. **パフォーマンス**: コード実行のより高速な起動時間
